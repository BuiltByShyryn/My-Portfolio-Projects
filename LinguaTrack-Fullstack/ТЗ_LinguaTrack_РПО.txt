Пример ТЗ в части РПО

Тема: Веб-платформа LinguaTrack для изучения английского и японского языков

Цель проекта:
Разработать веб-приложение для изучения языков, которое позволяет пользователям проходить курсы и уроки, выполнять тесты, отслеживать прогресс (стрик и XP), получать уведомления в Telegram, а также практиковать язык с ИИ-собеседником. Для преподавателя/администратора должна быть доступна панель управления контентом (создание/редактирование курсов, уроков и тестов).

Технологии:
Фронтенд:
• React (Vite)
• Tailwind CSS
• React Router
• Axios

Бэкенд:
• Django
• Django REST Framework
• JWT-аутентификация (djangorestframework-simplejwt)
• Telegram Bot API (pyTelegramBotAPI)
• Интеграция AI-провайдеров (Groq / OpenRouter / Gemini / OpenAI) + оффлайн-режим практики

База данных:
• PostgreSQL (основная БД)


Функциональные требования:

1) Регистрация и авторизация пользователей:
• Регистрация нового пользователя (username, email, пароль, опционально телефон и Telegram Chat ID).
• Авторизация существующего пользователя с использованием JWT (Access/Refresh).
• Обновление access-токена через refresh-токен.
• Выход из системы (очистка токенов на клиенте).
• Сброс/восстановление пароля (минимум как требование ТЗ; реализация может быть через email/код подтверждения или админ-команду в рамках демо).

2) Пользовательский профиль:
• Просмотр профиля: username, email, телефон, Telegram Chat ID, роль (student/teacher), streak, XP.
• Редактирование профиля: телефон, Telegram Chat ID (привязка), фото профиля.
• Просмотр статистики обучения: количество пройденных уроков, дата последнего занятия.

3) Курсы и уроки:
• Просмотр списка курсов.
• Фильтрация курсов по языку и уровню (например, EN/JP, A1/A2/B1).
• Просмотр детальной информации о курсе: описание, уроки, порядок уроков.
• Просмотр урока: теоретический материал, цель обучения, длительность.
• Поддержка изображений для уроков (URL или загруженное изображение).

4) Тестирование (Quiz):
• Прохождение квиза по уроку.
• Отображение вопросов и вариантов ответов.
• Подсчёт результата (score) и показ итоговой страницы.
• Хранение вопросов/ответов в базе данных.

5) Отслеживание прогресса (Progress tracking):
• Сохранение прогресса после завершения чтения материала и/или после прохождения квиза.
• Начисление XP за активность (например, за чтение и за тест).
• Поддержка streak (серия дней обучения): увеличение/сброс в зависимости от ежедневной активности.
• Получение прогресса по курсам для отображения на дашборде (процент прохождения).
• Получение «последнего прогресса» для быстрого продолжения обучения.

6) Telegram-уведомления:
• Отдельный Telegram-бот:
  – команды /start, /myid, /help;
  – выдача Chat ID пользователю для привязки в профиле.
• Отправка уведомления после завершения квиза (результат, streak, XP).
• Безопасное хранение токена бота в переменных окружения (.env), без публикации в репозитории.

7) AI Tutor (ИИ-собеседник):
• Чат с ИИ с сохранением истории диалога в базе данных (сессии и сообщения).
• Выбор языка интерфейса/общения (минимум EN/RU/KZ/JP для подсказок).
• Поддержка «Failover Strategy»: Groq → OpenRouter → Gemini → OpenAI → оффлайн-режим практики.
• Возможность прикреплять файлы в чат (например, .txt/.md/.pdf/.docx/.xlsx) — извлечение текста и добавление в контекст (в пределах лимитов).
• Показ пользователю, какой провайдер ответил (provider) и предупреждение, если включён оффлайн-режим.

8) Flashcards (карточки):
• Наборы карточек для микропрактики (термин, перевод/значение, пример).
• Возможность «пролистывания» и отметки знаний.
• (Опционально) добавление кастомных карточек преподавателем/пользователем.

9) Роли и панель преподавателя (+ Content):
• Раздел доступен только пользователям с ролью teacher/admin.
• CRUD для сущностей:
  – курсы;
  – уроки;
  – квизы;
  – вопросы;
  – (опционально) карточки.
• Механизм выдачи роли teacher:
  – через административную команду (например, manage.py make_teacher) и/или через админку Django.

10) Многоязычный интерфейс:
• Поддержка переключения языка интерфейса (например, KZ/RU/EN) на фронтенде.
• Отображение основных текстов/ярлыков в зависимости от выбранного языка.


Нефункциональные требования:

1) Безопасность:
• Пароли хранятся только в виде хэша (стандарт Django).
• Аутентификация через JWT (Bearer token).
• Защита от SQL-инъекций средствами ORM (Django ORM).
• Базовая защита от XSS на стороне фронтенда (экранирование, отказ от небезопасного HTML).
• Настройка CORS только для доверенных origins (локальная разработка и прод-адрес).
• Секреты (API-ключи, токены, SECRET_KEY, пароли БД) должны храниться в .env / переменных окружения и не публиковаться.

2) Производительность:
• Оптимизация запросов: select_related/prefetch_related на связанных сущностях (курс → уроки → квиз → вопросы).
• Пагинация списков (при росте объёма данных).
• Кэширование часто запрашиваемых справочных данных (опционально).

3) Масштабируемость и расширяемость:
• Возможность добавления новых языков и курсов без переработки архитектуры.
• Возможность подключения новых AI-провайдеров через единый интерфейс.
• Разделение приложения на модули (frontend компоненты / backend apps courses, users).

4) Надёжность:
• Грейсфул-деградация AI (оффлайн-режим практики при 429/quota/ошибках провайдера).
• Обработка ошибок API (единый формат ошибок, корректные HTTP-статусы).


Архитектура приложения:

Фронтенд:
• React SPA:
  – маршрутизация через React Router;
  – хранение токенов в localStorage (для демо), автоматическое добавление Bearer токена через axios interceptor;
  – компоненты: Dashboard, CourseCatalog/CourseView, UserProfile, TeacherPanel, AiTutor, Flashcards.

Бэкенд:
• Django REST API:
  – эндпоинты курсов/уроков/квизов;
  – эндпоинты прогресса и статистики;
  – эндпоинты профиля и привязки Telegram;
  – эндпоинты AI-чата (чат, история, удаление истории).
• Отдельный процесс Telegram-бота (bot.py), работающий параллельно с API.

База данных (логическая модель):
• Пользователь (CustomUser): роль, streak, XP, Telegram Chat ID, аватар.
• Course → Lesson → Quiz → Question.
• Progress: связь user + course + lesson + score + completed_at.
• AiChatSession + AiChatMessage: история AI диалогов.
